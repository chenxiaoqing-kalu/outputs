## Part B：示例/例题/习题（Worked Examples）

### EX-01：robot词源与命名构词

| 字段 | 内容 |
|------|------|
| **示例 ID** | EX-01 |
| **来源位置** | 发展背景 > 名词起源 |
| **示例类型** | 概念演示 |

#### 题目原文
"robot"一词最早广为人知的出处通常被认为是捷克作家卡雷尔·恰佩克在1920年前后发表的戏剧《R.U.R.》中使用的"robota"，在捷克语里意指"劳役、苦工"；"humanoid"词根"human-"与"人"相关，后缀"-oid"源自希腊语表示"类似……的形态/外观"。

#### 求解目标
理解"humanoid robot"与"人形机器人"的命名逻辑

#### 解题步骤
1. 拆解"robot"：robota（捷克语劳役）→泛化为"可替代人类执行任务的自动机器"
2. 拆解"humanoid"：human（人）+ -oid（类似……形态）= 外形或特征近似人类
3. 中文"人形机器人"：直译式构词，"人形"强调类人形态与运动机制，"机器人"强调自动化与可编程

#### 正确答案
"人形机器人"= 类人形态 + 可编程自动化机器，命名逻辑来自词根直译与行业约定俗成

#### 结构化条件
| 条件编号 | 条件内容 | 条件类型 |
|----------|----------|----------|
| C1 | robota源自捷克语，意指劳役/苦工 | 初始状态（词源） |
| C2 | -oid源自希腊语，表类似形态 | 约束条件（构词规则） |
| C3 | 20世纪中后期控制理论+机电+AI发展 | 过程变化（固定化为专业术语的驱动） |

#### 学科结构
| 维度 | 描述 |
|------|------|
| **核心逻辑关系** | 词源→词义演化→行业专业术语固定化 |
| **关键操作类型** | 词根拆解 + 历史语境还原 |
| **可变参数** | 其他科技术语（android、cyborg等）可做同类分析 |
| **结构不变量** | "形态特征+功能属性"双维命名结构 |

#### 题型特征标签
#术语词源分析 #构词逻辑 #科技词汇

### EX-02：人形结构与现有人类环境的匹配逻辑

| **来源位置** | 发展背景 > 设计动机 |
| **示例类型** | 概念演示 |

#### 题目原文
现实环境充满楼梯、门把手、开关、狭窄通道和以人手操作为标准的工具接口，若让机器人具备类人的身形、双足与双手，就能直接使用现有设施与工具，减少改造成本。

#### 求解目标
说明"环境兼容性"是人形设计的核心动机

#### 解题步骤
1. 识别现有环境的基准：人类建造的设施以人体尺寸和操作方式为标准
2. 推导形态选择：类人形态 → 可直接使用现有设施 → 减少改造成本
3. 补充动机：通用性（行走+精细操作）、协作安全（直觉友好）、终极目标（通用执行体）

#### 正确答案
人形设计动机的核心是：在无需改造现有环境的前提下，实现高通用性操作能力与人机协作效率

### EX-05：感知-认知-规划-控制-执行-反馈闭环

| 字段 | 内容 |
|------|------|
| **示例 ID** | EX-05 |
| **来源位置** | 工作原理 |
| **示例类型** | 模型分析 |

#### 题目原文
人形机器人的工作原理通常可以理解为一条"感知—认知—规划—控制—执行—反馈"的闭环链路：机器人先用传感器获取外界与自身状态信息，再把信息转化为可计算的环境模型与身体模型，随后生成动作计划，并通过电机与驱动器把计划落实为真实运动，最后依靠实时反馈不断修正误差。

#### 求解目标
理解六步闭环中每步的输入/输出及关键技术

#### 解题步骤
1. **感知**：视觉/深度/IMU/力觉 → 位姿、速度、接触状态估计（SLAM/iSAM2）
2. **认知**：维护环境表示（栅格/语义地图）+ 指令理解（大模型/VLA）
3. **规划**：全局路径规划（绕障）→ 步态/落脚点生成 + 操作规划（抓取姿态/全身协调）
4. **控制**：全身控制（WBC）/逆运动学/逆动力学 → 关节力矩/电流指令
5. **执行**：伺服电机+减速器+驱动器 → 机械输出；电源/BMS+实时通信总线
6. **反馈**：监测跌倒风险/关节过载/通信异常 → 降级策略/自恢复

#### 正确答案
六步闭环中纠错机制由"实时反馈"驱动，使机器人在复杂环境中持续稳定运行

#### 结构化条件
| 条件编号 | 条件内容 | 条件类型 |
|----------|----------|----------|
| C1 | 传感器类型：视觉/深度/IMU/力觉 | 初始状态 |
| C2 | 规划需同时满足稳定性、能耗、避障多约束 | 约束条件 |
| C3 | 执行层低延迟高可靠要求 | 约束条件 |
| C4 | 反馈实时监测异常触发降级 | 终止状态（安全机制） |

#### 学科结构
| 维度 | 描述 |
|------|------|
| **核心逻辑关系** | 顺序处理链 + 反馈闭环：各步输出作为下步输入，反馈层可中断并修正任意步 |
| **关键操作类型** | 闭环系统分析 + 各节点输入/输出/方法映射 |
| **可变参数** | 不同类型传感器组合、不同规划算法 |
| **结构不变量** | 感知→决策→执行→反馈的闭环结构不变 |

#### 题型特征标签
#闭环控制 #系统链路分析 #嵌入式系统

### EX-09：人形机器人面临的多维挑战体系

| 字段 | 内容 |
|------|------|
| **示例 ID** | EX-09 |
| **来源位置** | 面临挑战 |
| **示例类型** | 模型分析 |

#### 题目原文
人形机器人面临的挑战包括：技术挑战（运动不协调/感知弱/语言理解不准/决策机械化）、伦理与社会挑战（情感操控/数据隐私/就业替代/社会适应性）、成本与经济挑战（高研发制造成本/价格高/商业化难）、法律与监管挑战（责任归属/安全标准）。

#### 求解目标
构建四类挑战的结构框架，并识别各类的根本矛盾

#### 解题步骤
1. 技术挑战：根本矛盾 = 多自由度实时控制 vs 现有算力/传感器精度的差距
2. 伦理社会：根本矛盾 = 机器行为影响人类情感/劳动 vs 尚无监管框架
3. 成本经济：根本矛盾 = 高复杂度研发成本 vs 市场普及所需的低价格门槛
4. 法律监管：根本矛盾 = 机器自主行为 vs 现有法律主体为"人"的归责框架

#### 正确答案
四类挑战相互交织；技术挑战是基础矛盾，伦理/法律挑战随技术成熟而凸显

| C3 | 规模部署 → 就业/监管问题全面爆发 | 终止状态 |
